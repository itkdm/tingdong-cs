# Project Rules

- Before changing styles, confirm that every color is expressed through semantic theme tokens.
- Build new visual work for multi-theme support from the start. Add or extend theme tokens instead of hard-coding one-off colors.
- Prefer complete, extensible changes over tiny overrides when the current structure is the real problem.
- Split data, layout, styles, and behavior into focused files. Avoid pages that mix large content objects, page markup, and long CSS blocks.
- Write all visible website copy for visitors and learners. Avoid developer-facing roadmap language in the public UI.
- Keep the default theme as `tactical-board` until a new theme system is intentionally introduced.
